# 02 - Contraintes – Projet (RE)Sources Relationnelles

## Contraintes techniques
- Architecture logicielle basée sur le modèle **MVC (Modèle-Vue-Contrôleur)**.  
- Compatibilité navigateurs : Chrome, Safari, Firefox, Edge.  
- Développement en technologies web modernes (HTML5, CSS3, JS, framework web).  
- Base de données relationnelle (ex : PostgreSQL / MySQL).  
- Sécurité et anonymisation des données sensibles.

## Contraintes organisationnelles
- **Budget maximal** : 90 000 €.  
- **Délai de livraison** : 12 mois après la signature du marché.  
- Coordination entre les équipes projet (MOA, MOE, prestataires externes).

## Contraintes légales et réglementaires
- Respect du **RGPD** pour la protection des données personnelles.  
- Respect du **Référentiel Général d’Amélioration de l’Accessibilité (RGAA)**.  
- Conformité aux standards web (XHTML, CSS, W3C).

## Contraintes de qualité
- Interface utilisateur simple et accessible.  
- Performance et fiabilité du système.  
- Traçabilité et auditabilité des actions (journalisation).  
- Maintenance et évolutivité prévues.
